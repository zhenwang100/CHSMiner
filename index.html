      General help for CHSMiner (1.1)

      CHSMiner provides an integrated GUI environment for the detection, evaluation and visulization of chromosomal homologous segments (CHSs).

      Biological considerations

      CHS can be identified by genomic regions that contain a set of colinear homologous genes (Fig 1A). A limited number of unmatched genes could be allowed in this context because of extensive deletion and insersion. However, some studies have shown that conserved gene order may be too strict for ancient divergence (e.g. whole genome duplication in early vertebrate evolution), as gene inversion is another dominant force to dengenerate the genome. In order to fully exploit potential homology, CHSMiner define CHS based on shared gene content alone (Fig 1B). You can see the review of Simillion et al. [1] for more details.

      Figure 1 Graphical display of CHS

      Homologous genes at two genomic locations are labelled with the same color and linked.

      Features

      • Define CHSs based on shared gene content alone to fully explore potential homology.
      • Uniform design for both syntenies (between two different genomes) and paralogons (within the same genome).
      • Fast greedy algorithm to search for CHSs and formal statistical evaluation of the results.
      • Interactive visualization of CHSs from an individual one to whole genome pattern.
      • Entire Java package and convenient windows interface for end users.

      Note: CHSMiner is particularly useful to identify ancient CHSs that cannot be recognized from primary sequences.

      Getting started

      CHSMiner was entirely written in Java and complied in a jar package. Computers with Java 1.5 or higher installed are the only necessary condition to run the application. You can download the latest Java Runtime Environment (JRE) or or Java Development Kit (JDK) from http://java.sun.com/javase/downloads.

      You can start the application by either running "java -jar CHSMiner.jar" from the command line, or just double-clicking the jar file (Fig 2).

      Note: It is recommended to run the application by "java -Xms256M -Xmx256M -jar CHSMiner.jar", which will allocate more memory for it.

      Data input

      For user convenience, CHSMiner provides two ways for data input:

      Read data from a file

      Click the button to read data from a file.

      Below is an example of the CHSMiner format, including the name and type of each column. Every line is a gene record with items separated by Tab. The file can also contains titles and other annotations beginning with "#". There is no restriction for the order of the records, as the programme will automatically set ranks to them in preprocessing.

      # Gene ID Family ID Symbol Species Chromosome Orient Start End
      String String String String String String Integer Integer
      104 8280 ADARB1 Homo sapiens 21 + 45318942 45470901
      5930 5039 RBBP6 Homo sapiens 16 + 24458408 24491684
      110532 8280 Adrab1 Mus musculus 10 - 76753477 76881017
      ... ... ... ... ... ... ... ...

      "Start" and "End" are the physical positions (bp) of a gene on the chromosome. If a gene does not belong to any family, you can specify its family ID to be zero.

      You can download the above information for whole sequenced genomes from public databases of orthologs (e.g. HomoloGene, http://www.ncbi.nlm.nih.gov/sites/entrez?db=homologene ) or paralogs (e.g. EPGD, http://epgd.biosino.org/EPGD). We have prepared two sample data sets for test here: orthologs and paralogs. You can also download the data from the Ensembl database (see below for details).

      Download data from Ensembl database

      Click the button and then choose two species you want to compare.

      We prepared some of the species whose genomes have been completely assembled in Ensembl database (http://www.ensembl.org). CHSMiner will automatically fetch the information through its MySQL client interface. The speed is determined by your networks. If you want to fetch the data for the species you are interested in, you can connect the Ensembl public MySQL server (see http://www.ensembl.org/info/data/mysql.html), choose the ensembl_compara_xx database (where xx is the version), and execute the SQL query

      SELECT stable_id, family_id, display_label, taxon_id, chr_name, chr_strand, chr_start, chr_end
      FROM member, family_member
      WHERE member.member_id = family_member.member_id AND source_name = 'ENSEMBLGENE' AND taxon_id = ?

      You can find the taxon_id of your species in the NCBI Taxonomy database (http://www.ncbi.nlm.nih.gov/taxonomy/).

      Click the button to close the data set first if you want to change for a new input.

      Preprocessing

      Click the button for preprocessing. In the subsequent dialog box, you may have your options for the three tasks below:

      • Filter chromosomes you do not want to compare in this analysis. For example, you can remove those incomplete contigs from your data set.
      • Decide whether tandem duplicates should be merged. This should usually be yes as tandem duplicates will result in redundant information for the homologous regions.
        Note: In this software, we only merge the tandem duplicates without any interrupting genes. The suffix "_cluster" will be added to the symbols of the merged genes. You can also define other criteria to merge the tandem duplicates before you input the data.
      • Set null family ID, which indicates a gene does not belong to any family. The default value is zero.

      CHS detection and evaluation

      Click the button to start to search for CHSs.

      In the subsequent dialog box, in addition to deciding whether syntenies or paralogons should be identified, two critical parameters must be specified:

      • The size of a CHS, i.e. the number of distinct families (not genes) linked in the CHS. The default value is two.
      • The gap allowed in a CHS, i.e. the number of unmatched genes between two neighboring matched ones on the same chromosome. The default value is 30. According to our experience, this value is useful in general applications if whole genome data is available. However, you may need to adjust it for special data sets and purposes.

      The algorithm is carried out in two stages:

      • Compare each pair of chromosomes to search for all potential CHSs. A greedy algorithm is used in this stage, i.e. starting from two homologous genes, each at a different chromosomal location, and then looking for two other homologous genes that are each located within the predefined gap from the former two ones. The algorithm has only linear time complexity and is suitable for whole genome analysis.
      • Statistical evaluation of the resulted CHSs, i.e. caculating the P-value for a given CHS under the null hypothesis it is observed from two randomly permutated genomes. We implemented a theoretic algorithm based on the results of Hoberman et al. [2] rather than direct simulation to improve the performance and speed (see more details).

      The process may take a few seconds to minutes determined by your computer and data set. A progress bar will be shown for users to monitor this process.

      Output files

      The output table looks like this:

      # Species A Chromosome A Start A End A Species B Chromosome B Start B End B Size P-value
      # GeneID A Symbol A GeneID B Symbol B
      >Mus musculus 14 63616938 65074739 Homo sapiens 8 9949235 12220195 15 7.077E-55
      110265 Msra 4482 MSRA
      71037 4933401F05Rik 203074 UNQ9391

      The line beginning with ">" is a summary for a CHS:
      "A" and "B" are a pair of genomic locations that constitutes the CHS. "Size" is the number of distinct families (not genes) linked in the CHS. "P-value" is the statistical evaluation of the CHS under the null hypothesis that the gene orders of both genomes are random.

      Following the line, there is a list containing all the matched genes in the CHS.

      You can click the button in 'CHS result' pane to save the table as text format with items separated by Tab.

      CHS visualization

      CHSMiner can generate two types of images.

      Visualization of whole chromosomal pattern

      Click the button in 'CHS result' pane (Fig 3A) and select the chromosomes you want to compare in the subsequent dialog box. CHSMiner will render all CHSs between a given chromosome and the others (Fig 3B). The image is also interactive: you can browse CHSs on the chromosomal view, click on one and jump to the zoomed region (Fig 3C).

      Visualization of an individual CHS

      Select a CHS in the result table (Fig 3A) and then click the button . CHSMiner will render and label the homologous genes linked in the CHS (Fig 3C). The image can also be generated from chromosomal view (Fig 3B). For a large CHS, it may take a few seconds to render the image.

      When the CHS shown is too long to be clear, click the button in 'CHS visualization' pane to show only part of it (Fig 3D). You can specify the ranges in the subsequent dialog box (measured by gene ranks as you can capture in the image).

      Figure 3 Visualization of CHS

      Graphics operations

      Click the button in 'CHS visualization' pane to save current image. CHSMiner supports two common types of vector graph formats, i.e. Post Script (.ps) and Scalable Vector Graphics (.svg). These formats are convenient for further edit and can easily be converted to bitmap with high resolution.

      For user convenience, CHSMiner also provides some other simple graphics operations. Click the buttons and to browse all the images generated. Click the button to delete current image. Click the buttons and to zoom in and out the image. The image can also be dragged by your mouse for a better visualization.
